Privacy Policy

Claude RTL Fix — Chrome & Edge Extension

Last updated: 2026-05-14

Summary

Claude RTL Fix does not collect, store, transmit, sell, or share any personal data. The extension runs entirely in your browser and never sends information to any server.

What the extension does

Claude RTL Fix injects a CSS stylesheet and a small content script into pages on claude.ai. The script applies the standard unicode-bidi: plaintext CSS rule to message content and the input editor, which corrects the visual order of text when Hebrew (or other right-to-left languages) is mixed with English in the same message.

Data we collect

None. The extension does not collect, log, or transmit any of the following:

  • Personally identifiable information
  • Authentication credentials
  • Browsing history
  • The content of your conversations with Claude
  • IP address, location, or device identifiers
  • Analytics or usage statistics

Local storage

The extension uses the browser's chrome.storage.local API to remember a single preference: whether the RTL fix is enabled or disabled (the on/off toggle in the popup). This value is stored only on your device and is never transmitted anywhere.

Permissions explained

  • storage — to save your on/off preference locally.
  • Host permission for claude.ai — to inject the CSS fix into Claude conversation pages. The extension does not read, transmit, or modify the content of your conversations.

Remote code

The extension does not load or execute any remote code. All scripts and styles are bundled inside the extension package and reviewed by the Chrome Web Store before publishing.

Third parties

The extension does not contain any third-party trackers, analytics SDKs, advertising libraries, or external requests of any kind.

Children's privacy

The extension does not knowingly collect any data, so it does not collect data from children under 13 either.

Changes to this policy

If the data-handling practices ever change, this page will be updated and the "Last updated" date at the top will reflect the change. Material changes will also be reflected in the extension's listing on the Chrome Web Store.
